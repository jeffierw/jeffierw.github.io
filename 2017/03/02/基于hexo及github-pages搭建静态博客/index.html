基于hexo及github-pages搭建静态博客

发表于 2017-03-02 | 分类于 hexo | 阅读次数:

前言

  1. 本文只是本人搭建博客所踩过的坑，并不是教程。
  2. hexo和github好处很多，比如免费，不需要服务器，全是静态文件，也不用担心不安全，当然最好能在本地有备份。
  3. 域名不舒服可以自己随意绑定别的域名。

准备

  • 注册github.
  • 安装node.js, git.

github上的操作

  1. 新建一个名为“你的用户名.github.io”的仓库，这是必须的。

  2. 随便在哪鼠标右键点击”git bash here”,执行以下命令：

    $ cd ~/. ssh #嗯没提示是最好的

    ssh-keygen -t rsa -C "邮件地址" #这是注册github用的邮箱地址
    接着三次回车：生成文件“id_rsa.pub”，在用户(users)目录里找到”.ssh\id_rsa.pub”文件，记事本打开复制里面的内容。如果没找到pub文件，请设置显示隐藏文件。

  3. 进入github主页->SSH and GPG keys -> New SSH key：输入刚才复制的内容，title随便写。

  4. 测试是否本地连接github成功，命令行输入：

    $ ssh -T git@github.com

    提示Are you sure you want to continue connecting (yes/no)?输入yes回车，然后显示这个就是配置成功了：

    Hi jeffierw! You’ve successfully authenticated, but GitHub does not provide shell access.

hexo部分

  1. 安装hexo
    $ npm install -g hexo
  2. 在硬盘随便哪里建立一个文件最好起名叫hexo或者blog,我这里是在D:/hexo，然后分别执行：
    $ cd /d/hexo/
    $ hexo init
    hexo会自动下载许多文件到你建立的文件夹里。
  3. 修改一下主题，我用的是国内某大神letiantian基于国外某大神daleanthony修改的响应式主题，我很喜欢这种风格，推荐大家使用：
    $ git clone https://github.com/letiantian/huno.git
    然后进入根目录的-config.yml文件(这里必须使用sublime或vim等软件打开，因为可以更改utf-8格式，记事本打开修改内容容易乱码)，修改里面的theme: landscape为theme: huno,执行 hexo g重新生成。(这里又有坑，这个文件里面更改每一项前面都要有个半角空格)
    然后继续更改配置：
    deploy:
    type: git
    repository: git@github.com:jeffierw/jeffierw.github.io.git
    branch: master
    (这里的仓库输入自己的用户名，我这的是jeffierw)
    然后在命令行输入：
    $ npm install hexo-deployer-git --save
    安装这个东西。
    最后 hexo d -g 全部提交。

写博客

  1. 写博客工具我推荐sublime无需安装轻量化支持markdown语法
  2. 看一下简单的markdown语法。
  3. 每次写文章只需去d:/hexo/source/_posts下创建md文件完成后hexo d -g上传即可。创建文章也可以hexo new post "文件名"
