移动端之rem布局

发表于 2018-05-19 | 分类于 CSS | 阅读次数:

前言

之前的方案比如流式布局，flex布局无法做到：

  1. 页面布局文字不能随屏幕大小变化而变化。
  2. 流式布局和flex布局主要针对于宽度布局，但是无法对高度设置。
  3. 屏幕变化时元素高度和宽度无法等比例缩放。

rem基础

rem单位

rem(root em)是相对单位，类似于em，em是相对于父元素字体大小。

不同的是rem的基准是相对于html元素的字体大小。

比如，根元素（html）设置font-size=12px,非根元素设置width:2rem，换算成px表示24px。

rem的优点就是可以通过修改html的文字大小再经过媒体查询来整体控制页面中的元素。

媒体查询

媒体查询是CSS3新语法，@media可以针对不同的屏幕尺寸设置不同的样式。

mediatype表示媒体类型，值有all | print | screen

关键字and,not,only,表示将媒体类型或多个媒体特性连接到一起作为媒体查询的条件，其中not或者only表示排除某个媒体类型或者指定某个媒体类型，可以省略。

media feature表示媒体特性，必须有小括号包含，值有width | min-width | max-width,表示输出设备中页面可见区域的宽度，最小最大可见区域宽度。

媒体查询不同宽度设备时，习惯从小到大来写，这样因为CSS代码的层叠性，代码会更简洁。

引入资源

当样式比较繁多时，需要针对不同媒体使用不同的样式表，直接在link中判断设备的尺寸，然后引用不用的css文件。

less 基础

CSS是非程序语言，没有变量，函数，作用域的概念。

并且具有以下缺点：

  1. 需要书写大量看似没有逻辑的代码。冗余较高。
  2. 不方便维护和扩展，不利于复用。
  3. 没有很好的计算能力。
  4. 对非前端不友好，缺少CSS编写经验很难写出组织良好易于维护的CSS代码。

less(leaner style sheets)是一门CSS扩展语言，也叫CSS预处理器。

在现有的CSS语法上，为CSS加入程序语法的特性，引入了变量，Mixin(混入)，运算以及函数等功能，简化了CSS编写，降低了CSS维护成本。

常见CSS预处理器：Sass,Less,Stylus

基于node.js运行，全局安装npm install -g less

less编译

本质上，less包含一套自定义的语法和一个解析器，用户根据这些语法定义自己的样式规则，最终需要通过解析器，编译生成对应的CSS文件。

vscode有个好用的插件easy less,安装后，重新加载vscode，只要保存一下less文件，会自动生成CSS文件。

rem适配方案

  1. 让不能等比自适应的元素，达到当设备尺寸改变时，等比例适配当前设备。
  2. 使用媒体查询根据不同设备按比例设置html字体大小，页面元素使用rem作为尺寸单位，当html字体大小变化，元素尺寸也发生变化，从而达到等比缩放的适配。

rem实际开发适配方案

  1. 按照设计稿与设备宽度的比例，动态计算并设置html根标签的font-size大小（使用媒体查询）
  2. CSS中，设计稿元素的宽，高，相对位置等取值，按照同等比例换算为rem为单位的值（使用less等CSS预处理器）

设计稿常见尺寸宽度为640px,750px

其中元素大小取值方法：

  1. 页面元素的rem值 = 页面元素值（px） / （屏幕宽度 / 划分的份数）
  2. 屏幕宽度 / 划分的份数 = 根元素html font-size字体大小，即页面元素的rem值 = 页面元素值（px） / html font-size字体大小

例如淘宝将750px划分十份，即根元素字体大小为75px

rem适配方案技术使用（市场主流）

  1. 技术方案1：less等预处理器+媒体查询+rem
  2. 技术方案2（推荐）：淘宝flexible.js+rem
